# Contributing to VoidSlate

Thank you for your interest in contributing to VoidSlate.

## Before contributing

VoidSlate is licensed for non-commercial use only. By submitting a pull request, issue, patch, suggestion, design change, color update, documentation edit, or other contribution, you agree that your contribution may be included in VoidSlate under the same project license terms.

## Contribution rules

Please make sure that:

- Your contribution is your own work, or you have permission to contribute it.
- Your contribution does not include copied theme files, color palettes, assets, icons, fonts, code, or text from projects that are not compatible with VoidSlate's license.
- Your contribution does not include proprietary, confidential, or trade-secret material.
- Your contribution does not introduce telemetry, tracking, analytics, or data collection without a clear privacy update and maintainer approval.
- You use the project name **VoidSlate** consistently.

## Pull request checklist

Before opening a pull request, check that:

- The theme still loads in Visual Studio Code.
- Theme colors are readable across common editor states.
- New files include any required notices.
- Documentation changes mention **VoidSlate**, **MrakBook - Boris Karaoglanov**, https://mrakbook.com/voidslate, and boris@mrakbook.com correctly where relevant.
- The change does not permit commercial use or imply commercial permission.

## License of contributions

Unless there is a separate written agreement, all accepted contributions are included under the PolyForm Noncommercial License 1.0.0 and the notices in [`NOTICE.md`](NOTICE.md).

Contributors keep their copyright in their own contributions, but grant permission for those contributions to be used, modified, distributed, and maintained as part of VoidSlate under the project license.

## How to contribute

1. Fork the repository.
2. Create a branch.
3. Make your change.
4. Test the theme in Visual Studio Code.
5. Open a pull request with a clear description.
